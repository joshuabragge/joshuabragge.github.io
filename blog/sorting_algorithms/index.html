Simple Sorts

Selection Sort

Overview

Property value
Time Complexity Best: $O(n^2)$ Worst: $O(n^2)$
Space Complexity $O(1)$ in-place
Stability Stable Sort
Sort Type Comparision

Introduction

Find the smallest element from the array, exchange with the first and so on…

  • For every exchange, an element will be in its right place.
As seen in the figure below... if N is length of the array:
Total Comparisions: N^2/2
Total Exchanges: N

Pseudo Code


  public static void sort(int[] a) {
    int size = a.length;
    for(int i=0; i< size-1; i++) {
      int min = i;
      for(int j=i+1; j < size; j++) {
        if(a[min] > a[j]) min = j;
      //exchange i & min  
      }
    }
  }

Insertion Sort

Overview

Property value
Time Complexity Best: $O(n)$ Worst: $O(n^2)$
Space Complexity O(1) in-place
Stability Stable Sort
Sort Type Comparision

Introduction

For every iteration, take first element from unsorted part and insert it in its right place in sorted part.

Explanation

  • In first iteration, copy first unsorted element (2) into temp, move elements in sorted set to the right as long as you find a position for 2 to insert.
  • In second iteration, copy first unsorted (5) into temp, move 6 to the right to insert 5 in its right place.
  • In third iteration, copy first unsorted (1) into temp, move (6,5,2) to its right(blue arrows) to insert element 1 (temp) in its right place.
  • Continue so on until the operations complete.

Pseudo Code


public static void sort(int[] a) {
  int size = a.length;
  
  for(int i=1; i < size; i++) {
      int key = a[i];
      int j = i-1;
      while(j>0 && a[j] > key) {
          a[j+1] = a[j];
          j = j-1;
      }
      a[j+1] = key;
  }
}

Complexity Analysis

Selection Sort                              Cost            Times
public static void sort(int[] a) {                       
    int size = a.length;                     C1              1
    for(int i=1; i < size; i++) {            C2              n
        int key = a[i];                      C3              n-1
        int j = i-1;                         C4              n-1
        while(j>0 && a[j] > key) {           C5              $\sum _{i=1} ^{n-1} t_{i}$     
            a[j+1] = a[j];                   C6              $\sum _{i=1} ^{n-1} (t_{i}-1)$
            j = j-1;                         C7              $\sum _{i=1} ^{n-1} (t_{i}-1)$
        }
        //exchange i & min                   C8              n-1
    }
}

If n is the no of elements in the array…

$\mathbf{Time complexity} : T(n) = (cost\ of each\ statement) * (no\ of\ times\ its\ executed)$

$T(n) = C1*1 + C2*n + C3*(n-1) + C4*(n-1) + C5*(\sum _{i=1} ^{n-1} t_{i}) + C6*(\sum _{i=1} ^{n-1} (t_{i}-1)) + C7*(\sum _{i=1} ^{n-1} (t_{i}-1)) + C8*(n-1)$

The running time depends on $t_i$ and it varies according to the input.

Best case

  • In best case the statements inside while loop won’t execute since $a[j]<key$
  • $t_i=1$ for each i=0,1,2.. in for loop in line TBD while executing while loop.

so $\sum _{i=1}^{n-1} t_i = (n-1)\ \forall\ t_i = 1$

$T(n) = C1*1 + C2*n + C3*(n-1) + C4*(n-1) + C5*(n-1) + 0 + 0 + C8*(n-1)$

$T(n) = (C1-C3-C4-C5-C8) + n*(C2+C3+C4+C5+C8)$

$\approx O(n)$

Worst case

Lets take a decreasing sequence and try to sort it.

  • Now the while loop will execute $i$ times for each $i=1,2,3…n$

so $\sum _{i=1}^{n-1} t_i$ = $\sum _{i=1}^{n-1} i$ = ($\sum _{i=1}^{n})-n$ = $ (\frac{n(n+1)}{2})-n$

and

$\sum _{i=1}^{n-1} (t_i-1)$ = $\sum _{i=1}^{n-1} (i-1)$ = $(\sum _{i=1}^{n-1} i) - (\sum _{i=1}^{n-1} 1)$ = $ \Big(\Big(\frac{n(n+1)}{2})-n\Big) - \Big(n-1\Big)\Big)$

Substituting these in the original TBD equation, we will get $\approx O(n^2)$

Divide & Conquer

  • Divide the problem into many smaller subproblems
  • Conquer the subproblems by solving them recursively
  • Combine the solutions of these subproblems to get actual solution

Divide & Conquer approach comes under Recurrence relations. So we need to understand about recursion.

Let’s say we have a problem of size $n$, we recursively solve $a$ no of sub-problems each of size $n/b$, then we combine all these solutions in $O(n^d)$ time.

$T(n) = aT(n/b)+O(n^d)$

Merge Sort

Merge sort uses between 1/2 N lg N and N lg N compares to sort an array of length N

Time Complexity Analysis

$T(n) = \underbrace{T(\frac{n}{2})}_\text{left half} + \underbrace{T(\frac{n}{2})}_\text{right half} + \underbrace{n}_\text{merge 2 halfs}$

Take the left half and calculate

$T(\frac{n}{2}) = \underbrace{T(\frac{n}{4})}_\text{left half} + \underbrace{T(\frac{n}{4})}_\text{right half} + \underbrace{n/2}_\text{merge 2 halfs}$

Substitute 2nd equation in 1st

$T(n) = 2^2 * T(\frac{n}{2^2}) + 2 * n$

so on …….. For the last level.

$T(n) = 2^i * T(\frac{n}{2^i}) + i * n$

For the last level, it will be T(1). so $T(\frac{n}{2^i}) = T(1)$

Hence $\frac{n}{2^i} = 1 \iff n = 2^i \iff i=log_{2}n$

$T(n) = 2^{log_{2}n} * T(\frac{n}{2^{log_{2}n}}) + log_{2}n * n$

$T(n) = n + n log_{2}n$

$\approx O(nlogn)$
